# Access Request Process

**Owner:** IT Operations / Security
**Last updated:** 2026-01-25
**Applies to:** All employees and approved vendors

## When you need to file a request

Beyond your role's default access (provisioned automatically based on
your team and level at onboarding — see Onboarding Checklist: New
Hires), any access to a specific system, dataset, or warehouse beyond
your defaults requires an explicit access request. Common examples:
access to a specific customer's data in the CRM, access to a particular
warehouse's telemetry data if you're not on that warehouse's direct
support team, or elevated production database access.

## Data classification matters

The approval path depends on the Data Classification Policy level of
what you're requesting. Public and Internal-classified systems can
typically be self-approved by your manager. Confidential and Restricted
classified systems (which includes essentially all customer data and
production fleet control systems) require both manager approval and a
security review, which adds 1-2 business days to the process.

## Process

1. Submit the request through the IT portal, specifying the system,
   the specific access level needed (read-only vs. read-write vs. admin),
   and a business justification.
2. Manager approval — typically same-day if you're requesting access
   that's clearly relevant to your role.
3. For Confidential/Restricted systems, automatic routing to the Security
   team for review.
4. Access is provisioned and you receive a confirmation; for VPN-gated
   systems, you may need to reconnect to NorthLink for the new
   permissions to take effect — see VPN Setup Guide.

## Time-limited access

Some access grants (particularly for vendors, under Vendor Access
Policy, or for short-term project needs) can be requested as
time-limited, automatically expiring after a specified period rather
than requiring manual revocation. This is the preferred approach for any
temporary access need, since it removes the risk of access lingering
after it's no longer needed.

## Periodic access review

All Confidential and Restricted access grants are reviewed quarterly by
the Security team. If access hasn't been used in the prior 90 days, it
is automatically flagged for revocation unless the manager actively
confirms it's still needed.
